Usage & Limits

Learn how Sealmetrics tracks usage, what counts toward your limits, and how to manage overages.

Pageview Limits​

Each plan includes a monthly pageview allowance:

PlanMonthly Pageviews

What Counts as a Pageview​

Event TypeCounts?Notes
Page load✅Standard pageview
Virtual pageview (SPA)✅Single-page app navigation
Custom event❌Events are free
Conversion❌Conversions are free
Micro-conversion❌All events are free
API call❌API usage is separate

Example:

User visits 5 pages, adds item to cart, completes purchase

Pageviews counted: 5
Events (free): 2 (add-to-cart, purchase)
Total cost: 5 pageviews

Handling Overages​

What Happens When You Exceed​

When you exceed your pageview limit:

  1. Tracking continues - No data is lost
  2. Overage charges apply - €0.20 per 10,000 pageviews
  3. Notification sent - You're informed of the overage
  4. Invoice updated - Charges added to next bill

Reducing Usage​

Optimize Tracking​

If you're approaching limits, consider:

  1. Exclude internal traffic

    _sm('set', 'excludeIPs', ['192.168.1.0/24']);

  2. Filter bot traffic (enabled by default)

  3. Exclude staging/development

    • Don't install pixel on staging environments
    • Use separate test accounts

  4. Sample high-traffic pages

    // Track only 10% of homepage traffic
    if (Math.random() < 0.1) {
      _sm('pageview');
    }

Monitor by Account​

Identify high-usage accounts:

  1. Go to Settings → Usage
  2. View By Account breakdown
  3. Consider if all accounts need full tracking

FAQ​

Do events count toward pageviews?​

No. Only pageviews and virtual pageviews count. All events (conversions, micro-conversions, custom events) are free.

What happens if I hit my limit mid-month?​

Tracking continues and overage charges apply. You can set an overage limit to cap charges.

Can I buy extra pageviews without upgrading?​

Yes, through pageview add-on packs or automatic overage. Overage is simpler for occasional spikes.

Do unused pageviews roll over?​

No. Pageview allowances reset each billing cycle and don't roll over.

How quickly does usage update?​

Usage dashboard updates hourly. Real-time usage may have a ~1 hour delay.
