# Billing API

Manage subscriptions, invoices, usage, and checkout for your organization. These endpoints are a thin proxy that forwards cookie-authenticated dashboard requests to the internal billing microservice. The exact response shapes come from that service — treat the fields below as the contract surface and consult the live response for the canonical schema.

**Base path:** `/billing`

Scopes:
- `/billing/plans` is **public** (no authentication).
- `/billing/coupons/validate` and `/billing/checkout` accept `billing:checkout` OR `billing:manage` (the registration-flow token only has `billing:checkout`).
- Everything else requires `billing:manage` (or superadmin).

**Info:**
For the lightweight "is the org currently paying / blocked / trialing?" check, use [`GET /subscription/status`](./subscription). It's the endpoint the dashboard polls on every layout load.

---

## Plans

### List Plans

```http
GET /billing/plans
```

Public — no authentication required. Returns the catalog of plan tiers, prices, and feature flags currently offered.

---

## Overview & Invoices

### Get Billing Overview

```http
GET /billing/overview?org_id={org_id}
```

Returns the org's high-level billing snapshot (current plan, trial state, next renewal, payment method).

### List Invoices

```http
GET /billing/invoices?org_id={org_id}
```

Paginated list of past invoices for the org. Each invoice carries id, period, amount, status, and a PDF/hosted URL.

### Get / Update Accounting Email

```http
GET /billing/accounting-email?org_id={org_id}
PUT /billing/accounting-email
```

The email used to deliver invoices. The `GET` returns `{ email }`; the `PUT` body is `{ email: <string> }` (validated as a real email address).

---

## Subscription

### Get Subscription

```http
GET /billing/subscription?org_id={org_id}
```

Current subscription record (plan, status, billing interval, cancel_at_period_end).

### Cancel Subscription

```http
POST /billing/subscription/cancel?org_id={org_id}
```

Schedules cancellation at the end of the current paid period (does not refund the current period).

### Resume Subscription

```http
POST /billing/subscription/resume?org_id={org_id}
```

Reverses a scheduled cancellation while the period is still active.

### Subscription History

```http
GET /billing/subscription/history?org_id={org_id}
```

Audit trail of plan changes, cancellations, resumes, and upgrades.

---

## Usage

### Current Usage

```http
GET /billing/usage/current?org_id={org_id}
```

Events consumed in the current billing period plus the plan limit.

### Usage History

```http
GET /billing/usage/history?org_id={org_id}
```

Per-period historical usage rows.

### Overage Status

```http
GET /billing/overage-status?org_id={org_id}
```

Whether the org is currently in overage, the projected overage cost, and any forgive flag set by support.

## Checkout & Portal

### Stripe Customer Portal

```http
POST /billing/portal
```

Creates a one-time Stripe Customer Portal session. Used by the dashboard "Manage billing" button.

**Request Body:**

| Field | Type | Description |
|-------|------|-------------|
| `return_url` | string | URL Stripe redirects to after the portal session (max 500 chars) |
| `org_id` | string | Organization ID (max 100 chars) |

Response includes a `url` field — redirect the browser there.

### Validate Coupon

```http
POST /billing/coupons/validate
```

**Scope:** `billing:checkout` OR `billing:manage`. The registration flow uses this to apply promo codes before completing checkout.

**Request Body:**

| Field | Type | Description |
|-------|------|-------------|
| `code` | string | Coupon code (max 100) |
| `plan_tier` | string | Target plan tier (max 50) |
| `billing_interval` | string | `monthly` or `annual` (max 50) |

### Create Checkout Session

```http
POST /billing/checkout
```

**Scope:** `billing:checkout` OR `billing:manage`. Creates a Stripe Checkout session for first-time subscription purchase from the registration flow.

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `org_id` | string | Yes | Organization ID |
| `plan_tier` | string | Yes | Target plan tier |
| `billing_interval` | string | Yes | `monthly` or `annual` |

Response includes the Stripe Checkout `url` for browser redirect.

## Shopify Billing (Managed Pricing)

Shopify-installed accounts go through Shopify's **Managed Pricing** flow instead of Stripe. The three endpoints below sit under `/shopify/billing/*`:

- `GET /shopify/billing/redirect-handler` — public, handles the Shopify Admin redirect after a plan change.
- `GET /shopify/billing/pricing-page-url?shop={shop}` — computes the deep-link to the shop's Managed Pricing page in Shopify Admin.
- `GET /shopify/billing/status?org_id={org_id}` — current plan / charge state from the Shopify side; use this instead of `/billing/subscription` for Shopify-billed orgs.
