# API

REST API for querying analytics data programmatically.

| Specification | Value |
|---------------|-------|
| Base URL | `https://api.sealmetrics.com/api/v1` |
| Protocol | HTTPS only |
| Format | JSON |
| Authentication | API Key or JWT Bearer token |
| Rate Limits | Per-plan (see [Rate Limits](./rate-limits)) |
| Documentation | `/docs` (Swagger), `/redoc` (ReDoc) |

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/sites/{site_id}/stats` | Real-time summary stats |
| GET | `/sites/{site_id}/pageviews` | Pageview timeseries |
| GET | `/sites/{site_id}/visitors` | Unique visitors |
| GET | `/sites/{site_id}/pages` | Top pages with metrics |
| GET | `/sites/{site_id}/referrers` | Traffic sources |
| GET | `/sites/{site_id}/geo` | Geographic breakdown |
| GET | `/sites/{site_id}/devices` | Device analytics |
| GET | `/sites/{site_id}/conversions` | Conversion metrics |
| GET | `/sites/{site_id}/funnel` | Funnel report by UTM |
| GET | `/sites/{site_id}/live` | Live visitors (WebSocket) |
| GET | `/sites/{site_id}/settings` | Site settings |
| PATCH | `/sites/{site_id}/settings` | Update site settings |
| POST | `/sites` | Create new site |
| GET | `/health` | Health check |

## Quick Example

```bash
curl -X GET "https://api.sealmetrics.com/api/v1/sites/YOUR_SITE_ID/stats?period=7d" \
  -H "X-API-Key: str_your_api_key_here"
```

Response:

```json
{
  "period": "7d",
  "pageviews": 12543,
  "unique_visitors": 4521,
  "sessions": 5234,
  "bounce_rate": 42.3,
  "avg_session_duration": 185,
  "pages_per_session": 2.4,
  "conversions": 156,
  "revenue": 12450.00
}
```

## Response Headers

Every response includes rate limit information:

| Header | Description | Example |
|--------|-------------|---------|
| `X-RateLimit-Limit` | Max requests per minute | `60` |
| `X-RateLimit-Remaining` | Requests left in current window | `45` |
| `X-RateLimit-Reset` | Unix timestamp when limit resets | `1704067200` |
| `X-RateLimit-Tier` | Your plan tier | `starter` |

## Error Responses

All errors return JSON with this structure:

```json
{
  "error": {
    "code": "error_code",
    "message": "Human-readable description"
  },
  "request_id": "req_abc123"
}
```

| HTTP Code | Error Code | Description |
|-----------|------------|-------------|
| 400 | `bad_request` | Invalid parameters |
| 401 | `unauthorized` | Missing or invalid authentication |
| 403 | `forbidden` | Valid auth but insufficient permissions |
| 404 | `not_found` | Resource does not exist |
| 429 | `rate_limit_exceeded` | Too many requests |
| 500 | `internal_error` | Server error |
