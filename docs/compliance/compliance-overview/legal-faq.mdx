## **Data Collection & Calculation**

### **What data does Sealmetrics collect?**
We only collect four essential, non-personal variables:

- URL of the page  
- Referral URL  
- Timestamp  
- UTMs  
- User Agent (captured but **not stored**)  

All data is anonymous and isolated.

---

### **What data does Sealmetrics calculate?**
We compute:

- **Source ID** (`_adin`): Used for attribution
- **Session ID**: Temporary identifier that expires with the browser session

---

### **Does Sealmetrics use IP addresses for calculation?**
No.  
Sealmetrics **never uses, processes, or stores IP addresses** for any purpose â€” including bot filtering.

## **Bot Filtering**

### **How does Sealmetrics block bots?**
We use:

- **Bot user agent lists**, and  
- **Automated behavioral pattern detection**

IP addresses are **never used** for bot filtering.

---

## **Cookies & Storage**

### **Does Sealmetrics use cookies?**
No.  
Sealmetrics does **not** use:

- cookies  
- session storage  
- local storage  
- fingerprinting  
- cross-site IDs  

---

## **Real-Time Tracking & Retention**

### **Why doesnâ€™t Sealmetrics work in real-time by default?**
Because **real-time analytics legally requires user consent**, according to privacy regulators.  
To stay consentless and cookieless, real-time must be **explicitly activated** by the user.

---

### **How many months does Sealmetrics store my data?**
We store data for a **maximum of 24 months**.  
Regulators consider retention periods above 24 months as requiring consent.
