# Introduction to the Sealmetrics API

The Sealmetrics API is a powerful analytics and reporting interface that provides access to comprehensive marketing and sales data for your business. It allows you to programmatically extract metrics, integrate them into internal systems, automate reporting, and build advanced dashboards.

---

## Overview

The Sealmetrics API delivers detailed insights into:

- Marketing performance across all channels  
- Customer acquisition and attribution  
- Conversion and microconversion behavior  
- ROAS (Return on Advertising Spend)  
- Funnel progression  
- Segmented analytics using UTMs, countries, date ranges, and more  

All API responses follow REST principles and are returned in JSON format.

---

## Key Features

- **Comprehensive Marketing Data**  
  Access data from every marketing source, campaign, and time period.

- **Conversion Tracking**  
  Retrieve conversions and microconversions programmatically.

- **ROAS Analysis**  
  Measure revenue efficiency across paid channels.

- **Funnel Insights**  
  Understand where users drop off and where they convert.

- **Flexible Segmentation**  
  Filter reports using UTMs, dates, and other dimensions.

- **Pagination Support**  
  Efficiently fetch large datasets in multiple batches.

---

## Use Cases

You can use the Sealmetrics API to:

- Feed business intelligence tools with live marketing data  
- Build custom analytics dashboards  
- Automate weekly or monthly marketing reports  
- Combine Sealmetrics metrics with internal CRM or ERP data  
- Track cross-channel performance from a single source  
- Monitor conversion rates and customer journeys without cookies  

---

## API Version

This documentation refers to **API v1.0**, designed with:

- RESTful structure  
- JSON responses  
- Standard HTTP methods  

---

## Base URL

All endpoints begin with:

```
https://app.sealmetrics.com/api/
```

---

## Authentication

The API uses **Bearer token authentication**.

1. Obtain an access token via the `/auth/login` endpoint.  
2. Include the token in every request header:

```
Authorization: Bearer YOUR_TOKEN
```

---

## Standard Response Format

API responses follow a consistent structure:

```json
{
  "status": "ok",
  "data": [
    // results
  ]
}
```
