# Audit Logs

Audit Logs record significant actions taken across your Sealmetrics organization, providing a trail of who did what and when.

## Accessing Audit Logs

In the sidebar, go to **Organization → Audit Logs**.

Visibility is scoped to your organization: you see entries generated by members of the organization(s) you belong to. Access requires permission to manage members.

## What's Logged

Each entry is a single action with an actor, a resource it affected, and the time it happened. The actions that are recorded are:

### User

| Action | Meaning |
|--------|---------|
| User Created | A user account was created |
| User Updated | A user account was modified |
| User Deactivated / Reactivated | A user account was disabled or re-enabled |
| Login / Logout | A user signed in or signed out |
| Password Changed | A user changed their password |
| Email Verified | A user verified their email address |
| 2FA Enabled / Disabled | Two-factor authentication was turned on or off |
| Backup Codes Regenerated | 2FA backup codes were regenerated |

### Invitations

| Action | Meaning |
|--------|---------|
| Invitation Sent / Resent | A member invitation was sent or resent |
| Invitation Revoked | A pending invitation was cancelled |
| Invitation Accepted | An invited user joined |

### Accounts

| Action | Meaning |
|--------|---------|
| Account Created | A tracking account was created |
| Account Updated | Account settings were changed |
| Account Deactivated | An account was deactivated |

### Configuration

| Action | Meaning |
|--------|---------|
| Conversion Created / Updated / Deleted | A conversion definition changed |
| Content Group Created / Updated / Deleted | A content group changed |
| Settings Updated | A settings change was applied |

### API & Billing

| Action | Meaning |
|--------|---------|
| API Token Created / Revoked | An API token was issued or revoked |
| Plan Changed | The billing plan changed |
| Payment Method Updated | The payment method was updated |

:::note
There is no separate "failed login" event, and there is no logging of data exports, scheduled reports, or account archival — those actions don't exist as audit events.
:::

## What Each Entry Contains

Every entry stores:

- **Actor** — the user's name and email
- **Action** — one of the actions listed above
- **Resource** — the resource type and, when available, its ID
- **Timestamp** — when the action occurred
- **IP address** — the IP the action came from, when available
- **Details** — an optional JSON object with action-specific context

Entries do **not** record device, browser, geographic location, or before/after ("old value → new value") snapshots.

## Viewing the Log

The Audit Logs page shows a chronological list. Each row displays the actor, an action badge, the affected resource (type and ID when present), the timestamp, the IP address (when available), and any additional details captured for that action.

## Filtering

You can filter the list by **resource type** (User, Invitation, Account, Conversion, Content Group, API Token, Billing). The list is paginated.

There is no full-text search, no CSV export, no scheduled export, and no configurable audit alerts in the dashboard.

## Retention

Audit log retention is not configurable from the dashboard. There is no per-plan retention setting, export schedule, or self-serve deletion control. If you have specific retention requirements, contact support.
