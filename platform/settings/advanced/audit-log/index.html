Security Best Practicesâ€‹

Regular Reviewâ€‹

Set a schedule to review the audit log:

  • Daily: Check for failed logins, unusual activity
  • Weekly: Review user management actions
  • Monthly: Full audit review for compliance

Investigating Incidentsâ€‹

When investigating a security incident:

  1. Identify timeframe - When did the issue occur?
  2. Filter by time - Review all activity in that window
  3. Identify actors - Who was active during that time?
  4. Trace actions - Follow the sequence of events
  5. Check IPs - Look for unusual IP addresses
  6. Export evidence - Download CSV for records

Compliance Use Casesâ€‹

GDPR Data Access Requestsâ€‹

Track who accessed what data:

  1. Filter by "Data Actions"
  2. Search for specific user or data type
  3. Export for documentation

SOC 2 Audit Evidenceâ€‹

Demonstrate access controls:

  1. Export authentication logs
  2. Show user management history
  3. Document settings changes

Internal Security Auditâ€‹

Review account security posture:

  1. Check for shared credentials (multiple IPs for same user)
  2. Review inactive users
  3. Audit API key usage
  4. Verify 2FA adoption

Log Retentionâ€‹

Audit logs are available on Scale, Pro, and Enterprise plans.

PlanRetention Period
Scale90 days
Pro1 year
EnterpriseCustom

Logs are automatically deleted after the retention period. Export regularly if you need longer retention.

API Accessâ€‹

Query audit logs programmatically (Scale plan and above):

curl https://app.sealmetrics.com/api/audit-log \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "start_date=2024-01-01" \
  -d "end_date=2024-01-31" \
  -d "action_type=authentication"

Limitationsâ€‹

  • Audit logs cannot be modified or deleted
  • Bulk exports limited to 10,000 entries per request
  • Real-time streaming not available (use webhooks for real-time)
  • Some automated system actions not logged (internal processes)
